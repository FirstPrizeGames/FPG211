Account

선택적 Google 로그인을 추가했습니다.

Firebase Authentication의 Google 계정 선택 창을 사용하며 사이트 자체 비밀번호 입력란은 두지 않았습니다.

Usage

로그인 중에만 사용량을 계산합니다.

로그인 상태에서 페이지가 화면에 보이는 동안만 한도가 줄어들며 기록은 현재 브라우저에만 저장됩니다.

Access

Pro 결제 전에 로그인을 확인합니다.

Pro Checkout은 로그인 후에만 열리고, Team은 기존 예약 문의 흐름을 유지합니다.

Reliability

Status에 실시간 확인을 추가했습니다.

상태 페이지가 health endpoint와 주요 경로 및 공통 자산의 응답을 확인해 현재 결과를 표시합니다.

Trust

개인정보 안내를 실제 동작에 맞췄습니다.

Google 로그인, 로컬 사용량, Cache Storage, Google Forms, Stripe Checkout의 처리 경계를 명시했습니다.

Storage

쿠키 설정 항목을 명확히 구분했습니다.

필수 사이트 기능과 브라우저 안내 확인 선택을 분리하고 광고 및 분석 쿠키를 사용하지 않는 현재 상태를 표시합니다.

Reliability

짧은 페이지의 하단 배경을 수정했습니다.

본문이 짧아도 화면 전체를 같은 다크 테마로 채우도록 모든 경로의 최소 높이와 루트 배경을 맞췄습니다.

Design

공통 테마와 버튼 스타일을 통일했습니다.

페이지별로 달랐던 배경, 패널, 버튼 높이와 상태 색상을 공통 디자인 토큰으로 정리했습니다.

Design

팝업 UI를 공통 형식으로 정리했습니다.

설정, 프로필, 쿠키, 경고 및 공유 팝업의 표면, 버튼, 닫기 컨트롤이 같은 규칙을 사용합니다.
